Hyperparameter Tuning using Logarithmic Marginal Likelihood¶

This notebook explains the theory behind the Marginal Likelihood (LM) functional and how it can be used to find the optimal hyperparameters. Part 1 explains the theory if the functional while Part 2 provides an example implemented using the sklearn package to show how good hyperparameters influence the predictive quality of a GP. After that the functional itself is computed and visualized in accordance to the underlying process. Part 3 will show how the size of the training set influences the functionals behaviour.

Part 1 - Hyperparameter Optimization¶

Before we talk about how to tune your hyperparameters we have to take a step back and take a look at how we set what our hyperparameters are in the first place. To do so we briefly talk about Model Selection. Model Selection includes all aspects of the model including the choice of the covariance and its hyperparameters. This will help to improve predictions and makes interpretation by the user possible. The prolbem is: A multitude of possible covariance function families exist and each of them have numerous hyperparameters. Different hyperparameters have to be compared within a family and a comparison across families has to be made. From this it is clear that the model selection process is open ended but Literatur provides three general principles for model selection

  1. Compute probability of the model given the data
  2. Estimate the generalization error (error on unseen test samples)
  3. Bound the generalization error

Note: training error is usually a poor proxy of the generalization error due to overfitting (model learning the data / noise in the training set).
The following two sections present different approaches to "solve" the model selection process. Afterwards a section dedicated to model selection for GP Regression follows. The sections involve a bit of probability theory and I assume that you are familiar with terms like marginalisation, Bayes'rule, likelihood and other terms from the field of probability theory.

Bayesian Model Selection¶

This will be a short outline of the main ideas in Bayesian model selection.
A common way to approach the problem is a hierarchical specification of the models. At level 1 we look at the parameters w which can be the parameters in a linear model. The second level are the hyperparameters $\theta$ which control the distribution of the parameters (e.g. weight decay in a neural network) and the highest level contains a discrete set of possible model structures $\mathcal{H}_i$. Inference takes place at one level at a time applying general rules of probability theory [MacKay, D. J. C. (1992b). Bayesian Interpolation. Neural Computation].

Level 1¶

At this level we look at the posterior over the parameters w given by Bayes'rule

\begin{equation*} p(\textbf{w} | y, X, \textbf{$\theta$}, \mathcal{H}_i) = \frac{ p(\textbf{y} | X, \textbf{w}, \mathcal{H}_i) p(\textbf{w} | \textbf{$\theta$}, \mathcal{H}_i) }{ p(\textbf{y} | X, \textbf{$\theta$}, \mathcal{H}_i ) } \end{equation*}

where $p(\textbf{y} | X, \textbf{w}, \mathcal{H}_i)$ is the likelihood of the observations given the input space, the parameters w and the selected model $\mathcal{H}_i$. The term $p(\textbf{w} | \textbf{$\theta$}, \mathcal{H}_i)$ represents the parameter prior given the selected model and its hyperparameters. It encodes the probability distribution of our knowledge about the parameters prior seeing the data. If we have specific knowledge this probability distribuition will be narrow. The posterior $p(\textbf{w} | y, X, \textbf{$\theta$}, \mathcal{H}_i)$ then combines all the information we have through the observation (using the likelihood term $p(\textbf{y} | X, \textbf{w}, \mathcal{H}_i)$) and our prior beliefs about the model and its parameters. It is normalized the so called marginal likelihood (also called evidence) Note that the evidence is independent of the parameters since they get marginalized. It is given by

\begin{equation*} p(\textbf{y} | X, \textbf{$\theta$}, \mathcal{H}_i ) = \int p(\textbf{y} | X, \textbf{w}, \mathcal{H}_i) p(\textbf{w} | \textbf{$\theta$}, \mathcal{H}_i) d\textbf{w} \end{equation*}

Level 2¶

Now it is possible to derive a new posterior for the hyperparameters $\textbf{$\theta$}$ using the marginal likelihood from level 1.

\begin{equation*} p( \textbf{$\theta$} | y, X, \mathcal{H}_i) = \frac{ p(\textbf{y} | X, \textbf{$\theta$}, \mathcal{H}_i) p(\textbf{$\theta$} | \mathcal{H}_i) }{ p(\textbf{y} | X, \mathcal{H}_i ) } \end{equation*}

where $p(\textbf{$\theta$} | \mathcal{H}_i)$ is the so called hyper prior (prior for the hyperparameters which control the parameter distribution). The normalizing constant is given analogously to level 1 but marginalizes of the hyperparameters $\textbf{$\theta$}$

\begin{equation*} p(\textbf{y} | X, \mathcal{H}_i ) = \int p(\textbf{y} | X, \textbf{$\theta$}, \mathcal{H}_i) p(\textbf{$\theta$} | \mathcal{H}_i) d\textbf{$\theta$} \end{equation*}

Level 3¶

On the highest level in the model selection process the posterior for the model is computed

\begin{equation*} p( \mathcal{H}_i | y, X, ) = \frac{ p(\textbf{y} | X, \mathcal{H}_i) p(\ \mathcal{H}_i ) }{ p(\textbf{y} | X) } \end{equation*}

where $p( \textbf{y} | X) = \sum_{i}^{} p(\textbf{y} | X, \mathcal{H}_i)p(\ \mathcal{H}_i )$.

Its worth noticing that this kind of model selecting requires the user to compute several integral which may not be analytically tractable. One can therefore use Markov Chain Monte Carlo (MCMC) methods. In practice the calculation of the normalizing constant of level 2 can be difficult which then affects solving the equation for $p( \textbf{$\theta$} | y, X, \mathcal{H}_i)$.
To avoid solving the equation for the hyperprior one may approximates $p(\textbf{y} | X, \textbf{$\theta$}, \mathcal{H}_i )$ w.r.t. the hyperparameters $\textbf{$\theta$}$, known as type II maximum likelihood (ML-II). This approximation comes at the risk of inducing the possibility of overfitting into the selection process. This risk even increases when we have a high dimensional hyperparameter space.
Its worth noting that the marginal likelihood, e.g. in the equation for the parameter (w) posterior, automatically incorpoartes a trade-off between model fit and model complexity which makes is so valuable.

Cross-validation¶

Cross-validation (CV) is a well known and widely used technique for model selection. The idea is to split the available data set into two disjoint sets. One of which is used for training the model, the other one for validation. The models performance on the validation set is used as estimate for the generalization error. In practice only a fraction of data is used in validation which may lead to performance with large variance.
To minimize the variance in the generalization k-fold cross-validation can be used. In that case the data is split into $\textbf{k}$ disjoint, equally sized subsets and validation is done on a single subset while training uses the union of the remaining $\textbf{k-1}$ subsets. The whole procedure is repeated $\textbf{k}$ times, each time with a different subset for validation. Thus a large fraction of the data used for training. This comes at a price: $\textbf{k}$ models must be trained Typical $\textbf{k} = [3, ..., 10]$.

Model Selection for GP Regression¶

Marginal Likelihood¶

As mentioned above, Bayesian principles require analytical intractable computations (integrals over parameter space) but GPs are an exception. Since GPs are non-parametric models it is not immediately clear what the parameters of a model are. In the section before we called w in the level 1 inference. Recap that the posterior of the parameters was given by \begin{equation*} p(\textbf{w} | y, X, \textbf{$\theta$}, \mathcal{H}_i) = \frac{ p(\textbf{y} | X, \textbf{w}, \mathcal{H}_i) p(\textbf{w} | \textbf{$\theta$}, \mathcal{H}_i) }{ p(\textbf{y} | X, \textbf{$\theta$}, \mathcal{H}_i ) } \end{equation*}

and the evidence defined as

\begin{equation*} p(\textbf{y} | X, \textbf{$\theta$}, \mathcal{H}_i ) = \int p(\textbf{y} | X, \textbf{w}, \mathcal{H}_i) p(\textbf{w} | \textbf{$\theta$}, \mathcal{H}_i) d\textbf{w} \end{equation*}

which can be expressed as
\begin{equation*} \log p(\textbf{y} | X, \textbf{$\theta$}) = \frac{1}{2} \textbf{y}^T K_{y}^{-1} \textbf{y} - \frac{1}{2} \log |K_y| - \frac{n}{2} \log 2 \pi \end{equation*} where $K_y = K_f + \sigma_n^2 I$ is the covariance matrix for noisy targets $\textbf{y}$

This equation contains three terms which can be interpreted. The first term $\textbf{y}^T K_{y}^{-1} \textbf{y}$ contains the observed data is used to control the data fit. The second term $\log |K_y|$ is a complexity penalty since it depends on the covariance function. The last term is a normalization constant.

Since $\log p(\textbf{y} | X, \textbf{$\theta$})$ is conditioned on the hyperparameters $\textbf{$\theta$}$ it can be maximized with respect to it. This can be achieved by using the partial derivatives of $\log p(\textbf{y} | X, \textbf{$\theta$})$ w.r.t. every hyperparameter $\theta_j$. The partial derivatives are defined as

\begin{equation*} \frac{\partial}{\partial \theta_j} \log p(\textbf{y} | X, \textbf{$\theta$}) = \frac{1}{2} \textbf{y}^T K^{-1} \frac{\partial K}{\partial \theta_j} K^{-1} \textbf{y} - \frac{1}{2} \text{tr} \big( K^{-1} \frac{\partial K}{\partial \theta_j} \big) \end{equation*}

The complexity of solving this equation is $\mathcal{O} (n^2)$ once $K^{-1}$ is known. Therefore gradient based optimizers can be used.

According to http://www.gaussianprocess.org/gpml/ there is no guarantee the LML does find the global optimum since it is not a convex function (https://drafts.distill.pub/gp/). In fact there can be multiple local maxima which for simple covariance functions are not a severe problem. But if there are multiple maxima they correspond to different interpretations of the data. The number of local maxima can be reduced if the number of training samples is increased but often results in huge plateaus as seen in the following example.

Part 3 - Size of the training set¶

This section will determine how the size of the training set changes the "peakiness" of the LML functional. As stated in http://www.gaussianprocess.org/gpml/ the LML will have a more distinct global optimum the more data we have for training. This will be examined in this section.
To do so we draw a function from a random kernel as ground truth, sample from the same kernel, illustrate the LML functional for the hyperparameters and the samples and finally predict with a trained GP. The prediction is then compared to the ground truth.
